Governance

Authority Before Execution

Execution-time authority enforcement with measurable outcomes.

Invariant ID: ABE-EXEC-001
Enforcement Point: core/executor.py
Tracing: Opik (execution.commit)

1. Executive Summary

This repository demonstrates a single enforced invariant:

If explicit authority is not present, valid, and in scope at execution time, the state transition must not occur.

Authority is enforced at the execution boundary. It is not inferred, reconstructed, or scored post-hoc.

2. What Is Being Proven

  • Execution fails closed without authority
  • Authority must be valid at runtime (not earlier)
  • Scope violations are deterministically blocked
  • Expired approvals are rejected
  • All outcomes are observable and reproducible

3. Enforcement Boundary

The executor is the only location where state change is permitted. All allow / deny outcomes are decided here. 

core/executor.py
  └─ enforce_authority(decision)
     ├─ missing authority  → deny
     ├─ expired authority  → deny
     ├─ scope violation    → deny
     └─ valid authority    → execute

There are no alternative commit paths.

4. Decision Contract

Governance is enforced through a typed decision object. Execution depends only on fields present at runtime. 

Decision
  - proposal (action, params, rationale)
  - authority (approved_by, scope, expires_at)
  - created_at

This prevents approval context from being lost across wrappers, tools, or demos.

5. Reproducible Proof

To reproduce the invariant: 

python3 -m demo.run_demo

The demo performs four execution attempts against the same proposal:

  1. No authority
  2. Valid scoped authority
  3. Out-of-scope authority
  4. Expired authority

6. Observable Outputs

Each execution attempt produces:

  • Structured allow / deny result
  • Decision-level evaluation metrics
  • Immutable JSON artifacts written at runtime
  • Execution boundary traces in Opik

7. Evaluation Verdict

Invariant ABE-EXEC-001: HELD

at the execution boundary.

8. Non-Goals (Explicit)

  • Identity or RBAC systems
  • Approval user interfaces
  • Policy DSLs
  • Multi-agent orchestration
  • Production deployment concerns

This dossier isolates enforcement, not scale.
